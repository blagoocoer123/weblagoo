# Version 6.0 - SECURITY UPDATE 🔒

## Критические исправления безопасности:

### 1. Content Security Policy (CSP)
✅ Добавлены мета-теги CSP для защиты от XSS атак
- Ограничен источник скриптов, стилей и медиа
- Защита от инъекций вредоносного кода
- Блокировка iframe embedding (защита от clickjacking)

### 2. Защита от Tabnabbing
✅ Все внешние ссылки теперь имеют `rel="noopener noreferrer"`
- YouTube, Discord, Steam, FunPay, Telegram
- Предотвращает доступ к window.opener
- Защита от фишинговых атак

### 3. Безопасные HTTP заголовки
✅ Добавлены защитные мета-теги:
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `X-XSS-Protection: 1; mode=block`
- `Referrer-Policy: strict-origin-when-cross-origin`

### 4. Защита от утечки информации
✅ Удалены все console.log с debug информацией
✅ Убраны inline скрипты (перенесены в js/title-animation.js)
✅ Добавлены aria-label для accessibility

### 5. Rate Limiting & API Security
✅ Добавлен rate limiting для счетчика просмотров:
- Cooldown 5 секунд между запросами
- Timeout 5 секунд для fetch
- Валидация ответов от API
- Защита от CSRF с заголовком X-Requested-With
- AbortController для отмены зависших запросов

### 6. Улучшенная защита контента
✅ Сохранена защита от копирования
✅ Блокировка DevTools горячих клавиш
✅ Отключение контекстного меню

### 7. Документация безопасности
✅ Создан файл SECURITY.md с:
- Рекомендациями по настройке Nginx/Apache
- Примерами безопасного API
- Инструкциями по мониторингу
- Списком инструментов для проверки безопасности

## Что НЕ защищено:
⚠️ Важно понимать, что защита от копирования не является 100% надежной. Опытные пользователи могут обойти JavaScript защиту через DevTools. Эти меры усложняют копирование, но не делают его невозможным.

## Рекомендации:
📋 См. файл SECURITY.md для настройки серверной части
🔍 Проверьте сайт на https://securityheaders.com/
🔒 Обязательно используйте HTTPS с HSTS
